# VPSKit v0.6.2-beta Notes

## Status

`v0.6.2-beta` adds Trojan credential rotation and redacted export support while keeping the existing VLESS Reality TCP 443, Hysteria2 UDP 443, and Trojan TCP 8443 layout unchanged.

## Included

- `vpskit rotate trojan`
- `vpskit rotate trojan --yes`
- `vpskit rotate trojan --dry-run`
- Redacted Trojan export mode with `vpskit sub export trojan --redact`
- Redacted file export with `--redact --output <path>` and `--redact -o <path>`
- Sensitive output protection so verify and doctor no longer need to expose Trojan passwords
- Rollback behavior that restores the previous Xray config and Trojan state files if rotation fails
- New credential-rotation docs in English and Chinese

## Non-Goals

- SaaS control plane
- Telegram Bot
- PayPal or billing automation
- Web UI
- QR code generation
- Docker
- trojan-go
- Any new protocol
- Any change to hardening semantics
- Any change to VLESS Reality TCP 443 semantics
- Any change to Hysteria2 UDP 443 semantics
- Any move away from Trojan on TCP 8443
- Any web dashboard or online subscription API

## Live Validation Checklist

Run these commands on a live VPS after rotation:

```bash
vpskit rotate trojan --dry-run
vpskit rotate trojan --yes
vpskit verify trojan
vpskit verify vless-reality
vpskit verify hysteria2
vpskit doctor
vpskit sub export trojan
vpskit sub export trojan --redact
systemctl status xray.service --no-pager
journalctl -u xray.service -n 80 --no-pager -l
xray run -test -config /usr/local/etc/xray/config.json
ss -H -ltnp 'sport = :8443'
ss -H -ltnp 'sport = :443'
ufw status verbose
stat -c '%U:%G %a %n' /etc/vpskit/trojan /etc/vpskit/trojan/server.crt /etc/vpskit/trojan/server.key
cat /var/lib/vpskit/trojan.yaml
```

## Operational Notes

- The rotation flow validates a candidate Xray config before it touches the live config.
- If post-validation fails, VPSKit restores the previous Xray config and Trojan state files, restarts Xray, and reports rollback status.
- The redacted export keeps the server, port, SNI, and `allowInsecure` visible while replacing the password with `REDACTED`.
- Do not share `/var/lib/vpskit/trojan.yaml` publicly; it contains the live password.
- Process-level checks still show `xray` on TCP 8443, which is expected.
