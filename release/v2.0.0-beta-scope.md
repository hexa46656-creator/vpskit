# VPSKit v2.0.0-beta Scope

Historical note: this document is superseded by the v0.3.x beta line after Phase 1 VLESS Reality stabilization. Keep it for audit context; use `release/v0.3.0-beta-notes.md` and `release/v0.3.0-beta-test-report.md` for current operator guidance.

## Included

- installer-facing CLI
- `install hardening`
- `install vless-reality`
- `status`
- `doctor`
- `sub`
- `fix`
- subscription generation and repair helpers
- Shadowrocket repair
- DNS health
- TCP probe
- fallback report
- safe diagnostics
- Ubuntu 24.04 LTS Phase 1 hardening
- Xray VLESS Reality over TCP 443 with `xtls-rprx-vision`
- safety gates for SSH port detection, authorized_keys verification, package preflight, and existing Xray config protection
- explicit managed-user SSH public key input through `VPSKIT_AUTHORIZED_KEY` or `VPSKIT_AUTHORIZED_KEY_FILE`
- active-UFW allow/reload for the configured VLESS Reality TCP port

## Rollback Limits

- File writes are transaction-backed where practical.
- Package installation, Linux user creation, sudo group changes, UFW state changes, and service restarts are not fully reversible automatically.
- Existing Xray configs are not overwritten unless `VPSKIT_XRAY_FORCE_OVERWRITE=1` is set.
- The official Xray installer may emit a systemd warning about the special user `nobody`; VPSKit does not rewrite the upstream service unit in Phase 1.

## Excluded

- Hysteria2 installer
- Trojan installer
- FastAPI production service
- Telegram Bot
- PayPal or billing automation
- Web UI
- autonomous AI control plane
- self-modifying systems
- formal/meta/Gödel/epistemic/observer layers
- full SaaS auto-payment
- production billing automation
- destructive runtime behavior
