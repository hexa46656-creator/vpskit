# VPSKit v0.3.0-beta Notes

## Status

`v0.3.0-beta` is the Phase 1 VLESS Reality beta baseline.

This release was validated on clean DMIT Ubuntu 24.04 VPS instances and confirmed with live Shadowrocket traffic accepted by Xray logs.

## Included

- `vpskit install hardening`
- `vpskit install vless-reality`
- `vpskit sub show`
- `vpskit doctor`
- Ubuntu 24.04 LTS support for Phase 1 install targets
- Xray VLESS Reality over TCP 443
- UFW `443/tcp` allow/reload when UFW is already active
- Explicit managed-user SSH key support through `VPSKIT_AUTHORIZED_KEY` and `VPSKIT_AUTHORIZED_KEY_FILE`
- Default subscription path alignment between installer, `doctor`, and `sub show`

## Operator Guidance

After hardening, keep the root SSH session open until `alex` is verified from a second terminal.

Recommended verification sequence:

```bash
bash vpskit/cli/vpskit.sh verify ssh-user
bash vpskit/cli/vpskit.sh verify vless-reality
bash vpskit/cli/vpskit.sh sub show
```

## Not Included

- Hysteria2 installer
- Trojan installer
- SaaS control plane
- Telegram Bot
- PayPal or billing automation
- Web UI

## Known Limitations

- The official Xray installer may emit a systemd warning about `User=nobody`; VPSKit does not rewrite the upstream Xray service unit in this beta.
- Rollback does not fully reverse apt installs, Linux user creation, sudo group changes, UFW state changes, or service restarts.
- Existing Xray config is protected and requires `VPSKIT_XRAY_FORCE_OVERWRITE=1` before replacement.

## Supersedes

The older `v2.0.0-beta` release documents are historical planning/release-candidate artifacts and are superseded for the current Phase 1 VLESS Reality beta line.
