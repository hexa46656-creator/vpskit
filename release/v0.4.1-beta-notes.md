# VPSKit v0.4.1-beta Notes

## Status

`v0.4.1-beta` keeps the existing VLESS Reality installer and improves client export UX for the subscription file that already exists on disk.

## Included

- `vpskit sub export raw`
- `vpskit sub export base64`
- `vpskit sub export shadowrocket`
- `vpskit sub export v2rayng`
- `vpskit sub export clash-meta`
- `vpskit sub export sing-box`
- `vpskit sub export <format> --output <path>`
- `vpskit sub export <format> -o <path>`
- `vpskit sub validate`
- Grep-friendly export status lines for file output
- Grep-friendly subscription validation lines for the installed VLESS Reality URI

## Non-Goals

- Hysteria2
- Trojan
- SaaS control plane
- Telegram Bot
- PayPal or billing automation
- Web UI
- QR code generation
- Any change to Xray install semantics
- Any change to hardening semantics
- Any installer architecture rewrite

## Known Limitations

- This release only improves how the existing VLESS Reality subscription is exported or validated.
- It does not add new protocol support.
- It does not create parent directories for `--output` paths.
- Exporting to an invalid path fails closed and reports a write failure or path problem.
- Validation only checks the installed VLESS Reality URI fields that the current installer emits.

## Operator Guidance

Use `--output` when you want the export written to a file, or run `sub validate` before distributing the current subscription to clients.

Recommended commands:

```bash
bash vpskit/cli/vpskit.sh sub validate
bash vpskit/cli/vpskit.sh sub export clash-meta --output /tmp/vpskit-clash.yaml
bash vpskit/cli/vpskit.sh sub export sing-box --output /tmp/vpskit-sing-box.json
bash vpskit/cli/vpskit.sh sub export base64
```
