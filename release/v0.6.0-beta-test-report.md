# VPSKit v0.6.0-beta Live Test Report

## Environment

- DMIT Ubuntu 24.04
- Existing VLESS Reality on TCP 443
- Existing Hysteria2 on UDP 443
- New Trojan on TCP 8443 using Xray

## First Failure

The first live Trojan install failed because Xray could not read the TLS private key:

```text
/etc/vpskit/trojan/server.key permission denied
```

Xray runs as a restricted runtime user, so the key ownership and mode had to match that runtime.

## Manual Recovery

The live recovery sequence was:

```bash
chown -R nobody:nogroup /etc/vpskit/trojan
chmod 750 /etc/vpskit/trojan
chmod 644 /etc/vpskit/trojan/server.crt
chmod 600 /etc/vpskit/trojan/server.key
systemctl restart xray
```

## Fix

- Added runtime user and group detection for Xray
- Added TLS key ownership handling so Trojan cert files match the service runtime

## Second Failure

The next live validation failed because the candidate Xray config temp file did not have a `.json` suffix:

```text
Failed to get format of /tmp/tmp...
```

## Fix

- Candidate config temp files now use a `.json` suffix before `xray run -test -config`

## Final Pass

Observed live validation markers:

```text
TROJAN_INSTALL=pass
VERIFY_TROJAN=pass
VLESS_REALITY_PRESERVED=pass
VERIFY_VLESS_REALITY=pass
VERIFY_HYSTERIA2=pass
TCP_8443_LISTENER=pass service=xray
UFW_8443_TCP=pass status=active rule=present
```

## Final Port Layout

```text
TCP 443  -> xray -> VLESS Reality
TCP 8443 -> xray -> Trojan TLS
UDP 443  -> hysteria -> Hysteria2
```

## Remaining Limitations

- Trojan is a compatibility fallback, not the primary recommendation
- Xray warns that Trojan is deprecated / not recommended
- Self-signed TLS requires `allowInsecure` or a trusted certificate
- External TCP 8443 reachability still depends on provider, firewall, and client network path
