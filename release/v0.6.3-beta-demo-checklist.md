# VPSKit v0.6.3-beta Demo Checklist

## Preflight

- Confirm the branch is `v0.6.3-beta-final-qa-demo-packaging`
- Confirm the installed system is already on the expected VPSKit baseline
- Confirm no secrets will be shared outside the private handoff

## Generate QA

```bash
vpskit qa --redact --output ./qa-report.txt
```

- Confirm the final line is `VPSKIT_QA=pass`
- Review `VERIFY_VLESS_REALITY`, `VERIFY_HYSTERIA2`, and `VERIFY_TROJAN`
- Review `DOCTOR`

## Build Demo Package

```bash
vpskit demo package --redact --output ./vpskit-demo-package
```

- Confirm the directory exists
- Confirm `qa-report.txt` exists
- Confirm `protocol-layout.txt` exists
- Confirm `trojan-redacted.uri` exists
- Confirm the package includes English and Chinese notes

## Review Before Delivery

- Read `security-notes.en.md` and `security-notes.zh.md`
- Read `client-import-notes.en.md` and `client-import-notes.zh.md`
- Confirm no private key files are included
- Confirm no full Trojan password is included

## Private Export If Needed

```bash
vpskit sub export trojan --output ./trojan-private.uri
```

- Keep the real export private
- Use the redacted export for screenshots and support

## After Delivery

- Re-run `vpskit verify vless-reality`
- Re-run `vpskit verify hysteria2`
- Re-run `vpskit verify trojan`
- Re-run `vpskit doctor`
- Record any provider or network restrictions separately
