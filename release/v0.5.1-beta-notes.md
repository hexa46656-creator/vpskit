# VPSKit v0.5.1-beta Notes

## Status

`v0.5.1-beta` stabilizes Hysteria2 after live testing of `v0.5.0-beta`.

## Included

- Safer Hysteria2 reinstall behavior
- Clearer UDP 443 ownership handling
- Stronger install pass criteria
- Verify diagnostics that distinguish `bound`, `not_bound`, `owned_by_other`, and `unknown`
- Doctor output for binary, config, service, listener, subscription, and UFW state
- Recovery docs for Hysteria2 operators
- Live validation report from the `v0.5.0-beta` rollout

## Non-Goals

- Trojan installer
- SaaS control plane
- Telegram Bot
- PayPal or billing automation
- Web UI
- QR code generation
- Any change to Xray VLESS Reality install semantics
- Any change to hardening semantics
- Any installer architecture rewrite

## Stability Improvements

- Re-running `vpskit install hysteria2` is intended to be safe on an existing Hysteria2 host.
- UDP 443 bound by Hysteria itself is treated as reinstall-safe.
- UDP 443 bound by another process fails clearly.
- Existing UFW 443/udp rules are treated as present, not as an error.
- Inactive UFW is skipped without being enabled.

## Live Validation Checklist

Run the following on a live Ubuntu VPS after installation or recovery:

```bash
vpskit install hysteria2
vpskit verify hysteria2
vpskit doctor
ufw status verbose
ss -H -lunp 'sport = :443'
systemctl status hysteria-server.service --no-pager
journalctl -u hysteria-server.service -n 80 --no-pager
```
