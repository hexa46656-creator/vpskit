# VPSKit v0.6.0-beta Notes

## Status

`v0.6.0-beta` adds Trojan TCP 8443 support while keeping the existing VLESS Reality TCP 443 and Hysteria2 UDP 443 layout unchanged.

## Included

- Xray-based Trojan inbound support on TCP 8443
- Self-signed TLS by default, with no domain, certbot, or Let's Encrypt requirement
- `vpskit install trojan`
- `vpskit verify trojan`
- `vpskit sub export trojan`
- `vpskit sub export trojan --output <path>`
- `vpskit sub export trojan -o <path>`
- Doctor diagnostics for Trojan binary, config, service, listener, subscription, and UFW state
- UFW 8443/tcp handling that skips inactive UFW and adds the rule when active
- Xray service user/group detection for Trojan TLS ownership, including the `nobody:nogroup` runtime case
- Rollback protection that restores the previous Xray config if Trojan install fails after start or post-validation
- Grep-friendly install and verify output for live VPS validation

## Non-Goals

- SaaS control plane
- Telegram Bot
- PayPal or billing automation
- Web UI
- QR code generation
- Any change to hardening semantics
- Any change to VLESS Reality TCP 443 semantics
- Any change to Hysteria2 UDP 443 semantics
- Any move away from the existing 443/443 split
- Trojan-Go or any alternate Trojan implementation
- Any installer architecture rewrite

## Operational Notes

- Trojan uses `/etc/vpskit/trojan/server.crt` and `/etc/vpskit/trojan/server.key` for the self-signed TLS material.
- Trojan state is written to `/var/lib/vpskit/trojan.yaml` and `/var/lib/vpskit/trojan.env`.
- Clients that enforce TLS validation may need to trust the self-signed certificate instead of using `allowInsecure=1`.
- TCP 8443 must be reachable externally for clients to connect.
- The Trojan subscription export emits a single `trojan://` URI that can be imported into compatible clients.

## Live VPS Validation Checklist

Run the following on a live Ubuntu VPS after installation:

```bash
vpskit install trojan
vpskit verify trojan
vpskit doctor
ss -H -ltnp 'sport = :8443'
ss -H -ltnp 'sport = :443'
ufw status verbose
journalctl -u xray.service -n 80 --no-pager
```

## Operator Guidance

- Re-running `vpskit install trojan` should be safe when the Trojan inbound is already managed by VPSKit.
- The installer preserves the existing VLESS Reality inbound and its 443 listener.
- Use `vpskit sub export trojan` to generate a client URI after installation or recovery.
