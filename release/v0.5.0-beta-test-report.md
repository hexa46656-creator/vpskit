# VPSKit v0.5.0-beta Live Test Report

## Environment

- DMIT Ubuntu 24.04
- VLESS Reality already using TCP 443
- Hysteria2 intended to use UDP 443

## What Failed First

- The first Hysteria2 server config used a scalar `auth` value.
- The live error was:

```text
'auth' expected a map or struct, got "string"
```

## Second Failure

- The installer reported success before the service crash became visible.
- This exposed the need for delayed service validation and UDP listener validation.

## Final Fix

- Server config now uses the required auth map:

```yaml
auth:
  type: password
  password: ...
```

- The installer waits for the service state to settle before reporting pass.
- The installer validates the UDP 443 listener after restart.
- The installer stops the failed service to avoid a restart storm.

## Final Pass

Observed live validation output:

```text
HYSTERIA2_INSTALL=pass
VERIFY_HYSTERIA2=pass
UDP_443_LISTENER=pass service=hysteria
UFW_443_UDP=pass status=active rule=present
HYSTERIA2_SERVICE=active
HYSTERIA2_UDP_443=bound
```

## Remaining Limitations

- Self-signed TLS requires client acceptance or `pinSHA256`
- UDP reachability depends on provider and network path
- No Trojan, SaaS, Telegram Bot, PayPal, Web UI, or QR code support
