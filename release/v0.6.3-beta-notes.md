# VPSKit v0.6.3-beta Notes

## Goals

- Make the existing three-protocol VPSKit easier to verify
- Add a read-only final QA command
- Add a redacted demo/customer handoff package
- Add customer-facing handoff docs for delivery and verification

## Non-Goals

- SaaS
- Telegram Bot
- PayPal or billing
- Web UI
- QR code generation
- Customer dashboard
- Traffic billing
- Online subscription API
- Docker
- trojan-go
- Any new protocol
- Any requirement for a domain, Cloudflare, or Let's Encrypt
- Any change to hardening semantics
- Any change to VLESS Reality TCP 443 semantics
- Any change to Hysteria2 UDP 443 semantics
- Any change to Trojan TCP 8443 semantics

## New Commands

- `vpskit qa`
- `vpskit qa --redact`
- `vpskit qa --output <path>`
- `vpskit demo package`
- `vpskit demo package --redact`
- `vpskit demo package --output <dir>`
- `vpskit demo package --force --output <dir>`

## QA Checklist

- Run `vpskit qa --redact`
- Confirm the report ends with `VPSKIT_QA=pass`
- Confirm the report includes `VERIFY_VLESS_REALITY`, `VERIFY_HYSTERIA2`, and `VERIFY_TROJAN`
- Confirm the report includes `QA_READ_ONLY=yes`
- Confirm the report includes `SENSITIVE_OUTPUT=redacted`
- Confirm the report does not expose a full Trojan password

## Demo Package Checklist

- Run `vpskit demo package --redact --output ./vpskit-demo-package`
- Confirm the package contains `qa-report.txt`
- Confirm the package contains `protocol-layout.txt`
- Confirm the package contains redacted Trojan export content only
- Confirm the package contains customer-facing English and Chinese notes
- Confirm the package does not contain private keys

## Live Validation Checklist

```bash
vpskit qa --redact
vpskit verify vless-reality
vpskit verify hysteria2
vpskit verify trojan
vpskit doctor
vpskit sub export trojan --redact
vpskit demo package --redact --output ./vpskit-demo-package
ss -H -ltnp 'sport = :443'
ss -H -lunp 'sport = :443'
ss -H -ltnp 'sport = :8443'
ufw status verbose
systemctl status xray.service --no-pager
systemctl status hysteria-server.service --no-pager
```

## Sensitive Information Rules

- Do not publish the live Trojan password
- Do not publish `/var/lib/vpskit/trojan.yaml`
- Do not publish private keys
- Redact logs before sharing them
- Redacted exports are preferred for support and screenshots

## Known Limitations

- No SaaS
- No Bot
- No PayPal
- No QR
- No automatic domain or certificate provisioning
- Trojan remains a compatibility fallback
- External reachability still depends on the provider, network, and client
