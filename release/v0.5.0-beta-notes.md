# VPSKit v0.5.0-beta Notes

## Status

`v0.5.0-beta` adds Hysteria2 support as a focused second-protocol milestone. VLESS Reality behavior stays unchanged.

## Included

- `vpskit install hysteria2`
- `vpskit verify hysteria2`
- `vpskit sub export hysteria2`
- `vpskit sub export hysteria2 --output <path>`
- `vpskit sub export hysteria2 -o <path>`
- Doctor output for installed Hysteria2 state, UDP 443 listener state, subscription state, and UFW state
- Grep-friendly install and verify status lines

## Non-Goals

- Trojan installer
- SaaS control plane
- Telegram Bot
- PayPal or billing automation
- Web UI
- QR code generation
- Any change to Xray VLESS Reality install semantics
- Any change to hardening semantics
- Any installer architecture rewrite

## Known Limitations

- Hysteria2 is intentionally limited to UDP 443.
- The installer uses self-signed TLS and does not require a domain name.
- The client export writes a minimal YAML config and relies on the stored subscription file.
- The live binary download target is configurable through `VPSKIT_HYSTERIA2_RELEASE_TAG`, `VPSKIT_HYSTERIA2_RELEASE_ASSET`, and `VPSKIT_HYSTERIA2_INSTALL_COMMAND`.
- `VPSKIT_TEST_MODE=1` keeps the install path testable without network access.

## Live VPS Validation Checklist

Run the following on a live Ubuntu VPS after installation:

```bash
vpskit install hysteria2
vpskit verify hysteria2
vpskit sub export hysteria2
vpskit sub export hysteria2 --output /tmp/vpskit-hysteria2.yaml
ufw status verbose
ss -ulpn | grep ':443'
systemctl status hysteria-server --no-pager
journalctl -u hysteria-server --no-pager -n 80
vpskit doctor
```

## Operator Guidance

Use `vpskit verify hysteria2` before publishing the generated subscription file. If the binary download source needs to change for a specific host architecture, override the release tag or asset name rather than editing the installer flow.
