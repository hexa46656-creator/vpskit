# VPSKit v0.6.1-beta Notes

## Status

`v0.6.1-beta` stabilizes Trojan as a compatibility fallback while keeping the existing VLESS Reality TCP 443 and Hysteria2 UDP 443 layout unchanged.

## Included

- Trojan client compatibility docs in English and Chinese
- Trojan recovery docs in English and Chinese
- v0.6.0-beta live test report documenting the Xray runtime-user and candidate-config fixes
- Hardened Trojan export validation with exact single-URI output and file-write behavior
- Trojan doctor diagnostics with compatibility-fallback metadata
- Release and README guidance that positions Trojan as fallback while VLESS Reality remains the primary recommendation

## Non-Goals

- SaaS control plane
- Telegram Bot
- PayPal or billing automation
- Web UI
- QR code generation
- Docker
- trojan-go
- Any new protocol
- Any change to hardening semantics
- Any change to VLESS Reality TCP 443 semantics
- Any change to Hysteria2 UDP 443 semantics
- Any move away from Trojan on TCP 8443

## Live Validation Checklist

Run these commands on a live VPS after installation or recovery:

```bash
vpskit verify trojan
vpskit verify vless-reality
vpskit verify hysteria2
vpskit doctor
systemctl status xray.service --no-pager
journalctl -u xray.service -n 80 --no-pager -l
xray run -test -config /usr/local/etc/xray/config.json
ss -H -ltnp 'sport = :8443'
ss -H -ltnp 'sport = :443'
ufw status verbose
stat -c '%U:%G %a %n' /etc/vpskit/trojan /etc/vpskit/trojan/server.crt /etc/vpskit/trojan/server.key
cat /var/lib/vpskit/trojan.yaml
```

## Operational Notes

- Trojan remains an Xray inbound on TCP 8443.
- Process-level checks will show `xray` on 8443, which is expected.
- The Trojan subscription export emits a single URI with one trailing newline.
- Clients may need `allowInsecure=1` because beta releases use self-signed TLS.
