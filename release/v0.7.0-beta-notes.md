# VPSKit v0.7.0-beta Notes

## Goals

- Add a single redacted client bundle export for customer handoff and tutorials
- Keep existing protocol semantics unchanged
- Reuse the existing QA and subscription export flows

## Non-Goals

- SaaS
- Telegram Bot
- PayPal or billing
- Web UI
- QR code generation
- Customer dashboard
- Traffic billing
- Online subscription API
- Docker
- trojan-go

## New Commands

- `vpskit sub bundle`
- `vpskit sub bundle --redact`
- `vpskit sub bundle --output <dir>`
- `vpskit sub bundle --redact --output <dir>`
- `vpskit sub bundle --force --output <dir>`

## Sensitive Information Rules

- Do not publish the full subscription URI
- Use redacted output for screenshots and support
- Do not include `.env` files, private keys, or `server.key`
- Trojan remains redacted in the bundle
- Logs may contain source IP addresses

## Live Validation Checklist

1. Confirm `vpskit sub bundle --redact --output ./vpskit-client-bundle` succeeds.
2. Confirm the output directory contains the manifest, QA summary, protocol layout, and import notes.
3. Confirm the bundle includes `subscriptions/trojan-redacted.uri` and does not expose the full Trojan password.
4. Confirm `manifest.txt` records `VPSKIT_BUNDLE_VERSION=v0.7.0-beta`.
5. Confirm `qa-summary.txt` was generated from the read-only QA flow.
6. Confirm `vpskit verify vless-reality`, `vpskit verify hysteria2`, and `vpskit verify trojan` still pass.
7. Confirm `vpskit doctor` still reports the installed service state correctly.

## Known Limitations

- No SaaS
- No Bot
- No PayPal
- No QR
- No automatic domain or certificate provisioning
- Trojan remains the compatibility fallback
- External reachability still depends on the provider, network, and client
- Some clients may require manual import adjustments
